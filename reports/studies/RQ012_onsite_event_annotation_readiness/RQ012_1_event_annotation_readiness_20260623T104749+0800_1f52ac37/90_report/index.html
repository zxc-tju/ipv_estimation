Overview And Scope Boundary

RQ012A asks whether an onsite event annotation workflow is ready to support later human annotation and a separately authorized RQ012B analysis. The current conclusion is narrow: the outcome-blind event universe, threshold rationale, automatic pilot health checks, blind-package text surfaces, annotation protocol, and merge-validation guardrails are prepared. The package remains blocked for human labels and final issuance dependencies.

The behavioural reference is outcome-blind behavioural evidence, not ground truth. No event class is treated as proof of IPV, and construct-proximal motif labels are kept out of primary endpoint status.

The pilot remains outcome-blind and does not read labels or event-IPV associations.
